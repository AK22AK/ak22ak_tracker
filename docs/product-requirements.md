# 产品范围

## 定位

第一版只服务单个使用者，不按商业产品设计。它在手机上持续回答四个问题：

1. 今天应该做什么？
2. 今天实际做了什么？
3. 身体对训练的反应如何？
4. 明天的计划是否需要调整？

核心能力采用通用 tracker 模型，膝关节康复是第一个模块。以后可以增加护肤等
模块，但第一版不会为未知需求增加多用户、社交、付费或运营能力。

## 第一版用户流程

### 今日计划

- 打开手机 PWA 后直接看到当天有效计划版本及任务。
- 每项任务都是待办选框，只能由使用者确认完成或跳过。
- 显示当日反馈是否完成、数据同步状态和待确认的调整建议。

### 执行记录

- 可按计划勾选，也可记录计划外训练。
- 训练表单记录结构化内容，并允许补充主观感受。
- Garmin 活动可以建议关联某个任务，但关联和完成状态都由使用者确认。

### 身体反馈

- 每天至少提醒一次结构化反馈；完成后弱化提醒，但仍允许追加反馈。
- 膝关节模块优先记录疼痛、肿胀、功能表现、训练后及 24 小时反应。
- 突发情况可以随时新增，不覆盖当天早先的记录。

### 动态计划

- 反馈和执行记录保存后，可请求 DeepSeek 生成结构化调整建议。
- 页面展示修改原因、影响日期和任务差异。
- 使用者接受后，软件执行变更并生成新计划版本；拒绝则保留建议与决定。
- 所有历史版本可查看和回滚。

## 初始数据范围

- 初始计划来自现有康复计划文档，导入时记录来源仓库、路径和提交版本。
- 计划开始日期由使用者确定。
- 第一版不补录开始日期之前的任务执行、身体反馈或 Garmin 历史数据。
- 医疗检查、诊断和医嘱仍以原始笔记为权威来源，不复制到公开代码仓库。

## 成功标准

- iPhone 主屏安装后可快速进入今日页面。
- 离线时仍能查看最近缓存计划并提交待同步记录。
- 重复同步不会产生重复事件或重复 Garmin 记录。
- AI、Garmin 或 GitHub 任一服务故障时，今日计划与本地记录仍可使用。
- 任何外部活动都不能未经人工确认自动完成计划任务。
- 计划调整可解释、可确认、可审计、可回滚。

## 非目标

- 医疗诊断或替代医生意见。
- 自动批准高风险训练调整。
- 汇总与康复无关的全部健康数据。
- 多用户、公开分享、社区或商业化能力。
