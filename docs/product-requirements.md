# 产品需求

## 文档职责

本文是 AK Tracker 的产品目标、用户旅程、第一版范围和成功标准的权威来源。系统
怎样实现见[系统架构总览](architecture/overview.md)，当前完成度和实施顺序见
[项目计划](project-plan.md)，界面规范见[产品与 UI/UX 设计系统](design/product-and-visual-system.md)。

## 定位

第一版只服务单个使用者，不按商业产品设计。它在手机上持续回答四个问题：

1. 今天应该做什么？
2. 今天实际做了什么？
3. 身体对训练的反应如何？
4. 明天的计划是否需要调整？

核心能力采用通用 tracker 模型，膝关节康复是第一个模块。以后可以增加护肤等
模块，但第一版不会为未知需求增加多用户、社交、付费或运营能力。

## 信息架构

受保护的应用区域固定包含四个顶层页面：

- `今日`：当天任务、实际执行、反馈和待处理事项。
- `日历`：未来计划与历史执行、反馈和异常上下文。
- `趋势`：周完成、症状、负荷和计划版本时间线。
- `设置`：计划、集成、同步、通知、导出和本机数据。

底部导航始终可操作并保留各页面状态。网络数据不得阻塞日期、Tab、折叠区和表单
等本地交互；详细规则见[客户端数据与导航](architecture/client-data-and-navigation.md)。

## 第一版用户流程

### 今日计划

- 打开手机 PWA 后直接看到当天有效计划版本及任务。
- 每项任务都是待办选框，只能由使用者确认完成或跳过。
- 显示当日反馈是否完成、数据同步状态和待确认的调整建议。

### 执行记录

- 可按计划勾选，也可记录计划外训练。
- 训练表单记录结构化内容，并允许补充主观感受。
- Garmin 活动可以建议关联某个任务，但关联和完成状态都由使用者确认。

### 身体反馈

- 每天至少提醒一次结构化反馈；完成后弱化提醒，但仍允许追加反馈。
- 膝关节模块优先记录疼痛、肿胀、功能表现、训练后及 24 小时反应。
- 突发情况可以随时新增，不覆盖当天早先的记录。

### 动态计划

- 反馈和执行记录保存后，可请求 DeepSeek 生成结构化调整建议。
- 页面展示修改原因、影响日期和任务差异。
- 使用者接受后，软件执行变更并生成新计划版本；拒绝则保留建议与决定。
- 所有历史版本可查看和回滚。

### 出差、突发和暂停

- 可为一个日期范围启用临时执行上下文，并按当天场地、器械和时间条件选择私有
  备选方案。
- Tracker 使用固定的计划时区；当前私人 Tracker 为 `Asia/Shanghai`，短期出差时
  不随设备时区自动改变“今天”或计划生效日期。设备时区不同时，页面明确提示当前
  计划日按上海时间计算。
- 临时选择不自动改写基础计划；记录实际采用的方案、完成情况和身体反馈。
- 返程或暂停结束时，页面回顾最后有效阶段并给出接续差异；确认后才生成新版本。
- 生病、急性症状或红灯不套用普通出差降级流程，而进入停止和重新评估状态。

### 阶段和结项评估

- 计划目标日期只开启评估，不自动判定完成。
- 评估记录目标完成、左右侧反应、力量/动作控制、负荷耐受和下一阶段意向。
- 根据私有规则进入维护、专项进阶、延长当前阶段或人工复评等分支。
- 中间的出差、暂停和多次降级影响有效训练周，不能只按日历周数推进。
- 系统根据实际完成和安全反馈计算有效训练周并提出“进阶、重复或暂停”建议，最终
  由使用者确认；计算结果不能自动改写未来计划。

### 提醒与趋势

- 页面内持续显示今日反馈、待同步、待确认和集成失效状态。
- 主屏推送和角标是用户主动授权的可选增强，不是完成核心流程的前提。
- 趋势页展示症状、完成、负荷和睡眠背景的时间关系，不生成医疗诊断或因果结论。

## 初始数据范围

- 初始计划来自现有康复计划文档，导入时记录来源仓库、路径和提交版本。
- 计划开始日期由使用者确定。
- 第一版不补录开始日期之前的任务执行、身体反馈或 Garmin 历史数据。
- 医疗检查、诊断和医嘱仍以原始笔记为权威来源，不复制到公开代码仓库。

## 成功标准

- iPhone 主屏安装后可快速进入今日页面。
- 离线时仍能查看最近缓存计划并提交待同步记录。
- 重复同步不会产生重复事件或重复 Garmin 记录。
- AI、Garmin 或 GitHub 任一服务故障时，今日计划与本地记录仍可使用。
- 日期和顶层 Tab 在慢网络下立即响应，未加载数据只影响对应内容区。
- 任何外部活动都不能未经人工确认自动完成计划任务。
- 计划调整可解释、可确认、可审计、可回滚。

## 非目标

- 医疗诊断或替代医生意见。
- 自动批准高风险训练调整。
- 汇总与康复无关的全部健康数据。
- 依据 Garmin、睡眠或趋势图自动给出医学结论。
- 多用户、公开分享、社区或商业化能力。
