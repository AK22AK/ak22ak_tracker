# ADR-0005：临时执行条件使用覆盖层

状态：已接受

## 决策

出差、器械受限、只有零碎时间和暂停等临时条件保存为有时间范围的执行上下文，
不会因为每天选择不同备选方案就创建计划版本。具体备选内容来自私有计划；公共
核心只保存稳定选项标识、范围、原因和实际执行。

当返程、恢复训练或阶段评估确实改变未来任务时，软件生成差异预览，用户确认后
创建新的不可变计划版本。

## 原因

临时条件描述的是“怎样执行今天”，计划版本描述的是“未来正式安排发生了什么
变化”。混在一起会制造大量版本、让 AI 建议过期，并难以判断有效训练周。

## 结果

- 需要 `execution_context`、暂停/恢复事件和接续决策模型。
- 日历同时展示基础计划与当天执行上下文。
- 有效训练周按实际完成与安全反馈计算，不按日历自动推进。
- 目标日期只触发评估，不自动结束 Tracker。
