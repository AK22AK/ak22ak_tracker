# ADR-0004：客户端应用壳与服务端状态缓存

状态：已接受

## 决策

受保护的应用区域使用持久共享 Layout 作为 App Shell。顶层导航和日期选择先更新
客户端状态；TanStack Query 管理远端读取与缓存，Dexie 管理跨重启的私人缓存和
离线写入队列。Server Component 只用于首次预取和静态框架，不再承载每次日期
选择或 Tab 切换。

## 原因

当前动态页面把 UI 提交绑定到身份校验和多轮数据库查询。生产环境的单次动态请求
已经接近或超过一秒，日期切换因此出现数秒无反馈。缩短数据库耗时不能保证离线或
慢网体验，必须从交互边界上消除网络依赖。

Next.js 共享 Layout 可以在导航时保持状态；客户端查询缓存提供去重、后台刷新、
取消和精确失效；IndexedDB 可以在 iOS PWA 中保存结构化离线数据。

## 结果

- 需要增加 Query Provider、稳定 Query Key 和聚合 API。
- 日期通过本地状态立即选择，URL 使用 History API 同步。
- 缓存与权威数据明确分层，退出登录必须清除用户缓存。
- 组件测试必须覆盖慢网络、迟到响应和离线重放。
- 服务器查询和部署区域仍需优化，但不再决定界面是否响应。
