# ADR-0001：公开应用与私有数据分离

状态：已接受

## 决策

应用代码仓库公开，真实运行数据放在独立私有仓库和私人 PostgreSQL。公共仓库
只保留通用代码、匿名测试和脱敏文档。独立数据私仓由 `zhengjing_notes` 中的稳定
康复追踪入口引用，作为个人知识体系中的结构化扩展。

## 原因

代码适合复用和公开审查，而健康数据、医疗记录、计划与凭证不能随源码公开。
拆分后，后端可以只通过 GitHub API 修改目标数据文件，不需要同步大型笔记仓库，
也不会干扰原始笔记在其他设备上的 Git/iCloud 工作流。

## 结果

部署必须同时配置数据库和私有仓库凭证；本地开发可在没有私人数据时运行空状态。
获授权的个人 AI 默认从 `zhengjing_notes` 进入，再沿引用读取数据私仓并合并上下文；
这一知识导航关系不要求应用克隆或改写笔记仓库。
