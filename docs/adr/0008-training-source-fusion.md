# ADR-0008：训练数据按来源融合并由使用者确认

## 状态

已接受，2026-07-19。

## 决策

AK Tracker 不要求使用者重复录入已经存在于其他训练工具中的数据，而是按来源权威
边界读取、融合并建议关联：

- AK Tracker 计划版本是康复计划与任务的权威来源。
- Garmin 是活动发生、时长、距离、配速、心率和基础睡眠的主要来源。
- 训记是力量训练动作、重量、组次、次数、RPE 和训练备注的主要来源。
- 使用者是疼痛、肿胀、主观感受以及任务是否完成的唯一权威来源。
- AI 只负责整理、匹配和提出调整建议，不是事实来源，也不能确认任务完成。

Garmin 和训记记录分别保存为带 provenance 和内容版本的外部记录，可以同时关联到
同一任务。系统只提出关联建议；使用者确认、修改或取消关联，并独立确认任务完成。
来源记录随后变化时进入需要复核状态，不能静默改写确认时保存的实际执行快照。

训记第一版只读训练数据。应用不读取饮食、身体数据或官方计划，不调用训练写回
接口。API Key 只从认证设置页进入服务端，验证后认证加密保存；公开文档、代码、
日志、客户端缓存和 GitHub 镜像均不包含密钥。

## 原因

使用者已经在训记中持续记录力量训练细节，在 AK Tracker 再逐项填写会造成重复劳动，
并让两个副本逐渐不一致。Garmin 能证明活动发生并提供生理指标，但手表记录的力量
动作、重量和组次不够可靠；训记正好补足这部分。按来源融合可以保留各自优势，同时
让 AK Tracker 专注于康复任务、身体反馈、安全和计划调整。

外部记录不能直接完成任务，因为“发生了一次训练”和“按康复要求完成某项任务”不
完全等价。保留一次人工确认能防止误匹配，也能处理计划外训练、动作替换和不完整记录。

## 影响

- 训记只读同步和关联确认作为当前康复闭环的前置纵向切片，先于最终任务卡视觉收口，
  避免围绕重复手工输入固化错误界面。
- 数据模型需要支持 provider、稳定外部 ID、内容 hash、来源版本、关联决定和待复核
  状态；同步和任务命令保持分离。
- 今日页以来源摘要和确认操作为正常路径，手工训练表单退为兜底。
- Provider 故障只影响对应来源区，不阻塞计划、反馈、安全规则和手工记录。
- 如果未来需要训记写回、其他数据类别或自动确认，必须另行评审权限、安全和用户
  控制边界，不能沿用本决策自动扩大范围。
