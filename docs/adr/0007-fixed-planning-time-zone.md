# ADR-0007：Tracker 使用固定计划时区

状态：已接受

## 决策

每个 Tracker 保存一个固定的 `planningTimeZone`。当前膝关节 Tracker 使用
`Asia/Shanghai`；短期出差期间，设备时区变化不会自动改变计划日、任务日期或计划
版本生效日。

事件同时保存绝对发生时间、记录时间、发生地设备报告的 IANA 时区和 UTC 偏移，
并按 Tracker 的计划时区派生 `localDate`。设备时区与计划时区不同时，界面明确提示。

## 原因

康复计划按连续训练日和 24 小时反应推进。若跟随设备时区静默切换，跨时区旅行可能
制造重复日期、跳过任务，或把同一次训练和次日反馈归到错误计划日。固定计划时区让
计划版本、历史查询和离线重放保持确定，同时保留实际发生地时间供解释。

## 结果

- Tracker 数据模型需要计划时区，事件 Schema 需要发生地时区和 UTC 偏移。
- “今天”、计划版本解析和反馈归属使用同一日期工具，客户端和服务端必须一致。
- 修改计划时区必须是显式设置操作，并预览受影响日期，不能由浏览器自动完成。
- 测试覆盖上海日期边界、UTC 跨日和设备处于其他时区的情况。
