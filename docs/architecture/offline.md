# 离线流程

## 文档职责

本文是 PWA 离线可靠性的权威架构说明，负责私人缓存、待同步命令、重放、冲突、
身份清理和 AI 不可用时的行为。客户端运行时边界见
[客户端数据与导航](client-data-and-navigation.md)，服务器写入和幂等规则见
[数据与同步](data-and-sync.md)。

离线包含两类不同问题：业务数据暂时无法上传，以及 AI 暂时无法调用。

## 业务数据离线

- PWA 预缓存不含私人数据、也不依赖动态 Next.js chunk 的公共离线启动壳。
- TanStack Query 管理运行期服务端状态；P2a 将今日计划、月摘要和单日详情按白名单
  快照到 IndexedDB，供同一身份离线只读。
- P2a 只保存 `TrackerSafetyPolicy` 的 `policyId + version + hash` 引用，不持久化规则；
  P2b 实现离线反馈时，才会通过独立评审决定规则执行所需的最小私人白名单。
- 训练勾选和身体反馈的待同步命令属于 P2b；P2a 离线时只允许浏览并明确提示联网
  后操作。
- 界面明确显示“仅保存在本机”“正在同步”“已同步”或“需要处理”。
- 启动、重新联网、页面重新可见、窗口获得焦点时尝试重放；用户也可以手动重试。
- 只有服务端确认后才删除本地待同步副本。

Background Sync 在主流浏览器中不是统一可用能力，特别不能假设 iOS 在 PWA 已
关闭时一定会上传。P2b 的命令队列必须依靠下次打开应用后的前台重放，不能把后台
同步当成交付前提。

## 缓存策略

- 公共离线启动壳及其固定 JS/CSS 使用 Cache Storage；同源导航采用 network-first，
  网络失败时只回退公共壳。在线返回的认证页面、RSC 和 API 响应均不写入缓存。
- 公共壳在没有服务端 session/RSC 的情况下，从 IndexedDB 的 `active-identity` 和
  同一身份白名单快照恢复今日与日历；没有有效身份或快照时只显示安全空态。
- 公共壳仅在浏览器确实离线时读取快照；在线直接访问公共壳会先返回原受保护路径，
  由服务器重新校验身份，不能用本机快照代替在线身份认证。
- 公共壳使用独立、自包含的最小显示契约，按查询种类校验身份、Tracker、日期范围和
  所有实际显示字段。任一今日、月摘要或单日详情快照损坏时，只显示对应安全空态，
  不能把损坏数据解释成零任务或零反馈。
- 已认证 API 响应不进入 Service Worker Cache。
- P2a 快照以不可变 GitHub 数字用户 ID、`trackerKey`、查询种类和日期范围隔离；
  命中时经 TanStack Query 恢复，并显示“离线缓存”、保存时间和只读状态。
- 计划版本不可变，可长期缓存；日详情和集成状态使用较短过期时间。
- Schema 或用户身份变化时执行显式迁移/清理，不能让上一个身份的缓存继续可见。

## 队列和冲突

- 每条本地写入拥有客户端 UUID、idempotency key、创建时间、尝试次数和错误代码。
- 同一任务的连续编辑按领域规则合并或按顺序重放，不能简单覆盖其他设备已写内容。
- 服务端返回当前对象和版本；客户端用响应更新 Query Cache，而不是无条件全页
  刷新。
- 可重试网络错误指数退避；校验、权限和版本冲突进入“需要处理”。

计划缓存和未来的离线命令属于敏感健康数据。第一版假设私人手机本身受系统密码/生物
识别保护，不把数据写入 localStorage，不在 Service Worker Cache 中缓存 API
响应。安全策略引用虽然不是凭证，仍属于私人领域数据，执行相同的身份隔离和退出清理；
退出登录或执行“清除本机数据”时清理 IndexedDB。

## P2a 明确不包含的能力

- 不创建或重放离线任务、反馈和关联命令。
- 不声称“已保存到本机”或“已同步”；写控件在快照模式下提示联网后操作。
- 不缓存集成凭证、API Key、会话、Cookie、Authorization、Provider raw payload 或
  完整私人安全规则。
- 离线启动壳不声称已登录；恢复网络后重新进入受保护页面并由服务器校验身份。

## AI 离线

DeepSeek 无法离线运行。离线反馈照常保存，但不会伪造 AI 建议：

1. P2a 只显示已缓存反馈的权威安全结果，不在离线状态生成新反馈。
2. P2b 若实现离线反馈，必须使用同版本的私人确定性规则，并继续让服务端结果成为
   最终权威。
3. 反馈上传 PostgreSQL 成功后，才创建 AI 分析任务。
4. AI 建议返回后显示为“待确认”，绝不在后台静默改计划。

这样网络故障不会丢失反馈，也不会让旧建议误装成当前建议。

## 验收场景

- 飞行模式重启应用，仍能查看同一数字身份最近缓存的今日计划和日历，并看到保存
  时间与只读提示。
- 过期、损坏或其他身份的快照不得恢复。
- 离线任务、反馈和来源关联不得伪装成可保存。
- 恢复网络后重复触发同步不会制造重复事件。
- 同步期间关闭应用，下次打开继续处理剩余队列。
- 退出登录后无法从页面、IndexedDB 或 Cache Storage 读到私人数据。
